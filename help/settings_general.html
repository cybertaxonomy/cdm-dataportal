Drupal's CDM Dataportal module dialogues with the EDIT CDM Server asking for taxa data and classification of taxa. The first thing the CDM Dataportal module needs to know is where is the CDM Server located in order to access it and this parameters are configurable via the general settings here explained.

  • CDM web service URL

    This is the ip address of the location of the CDM Web Server which contains your collection database and it is accessible through web services. The first part of the address is the ip address asigned to the device/server where your CDM Server is running, the second part after the two points is the port where the CDM Server is listening the Web Services request. And the last part after the slash is the name of the database of your collection.

    Probably you will need to ask for are these parameters to your system adminitrator.

  • Rank of highest displayed taxon

    It is possible to navigate the taxa through a tree structure, here you can select the which should be the maximal rank to show at the top of the tree(s).

  • Enable caching

    This feature save the visited single taxon sites in an internal drupal cache doing the portal response of taxa sites faster beacause the sites are loaded from the cache and are not created from scratch.

    If taxa are modified by the editor or any other application the changes will be not visible till the cache is erased. Therefore developers should deactived this feature when they are working with the portal

  • Debug CDM Web Service

    This is a useful feature for developers. Activating this feature is possible to see which web services have been called and its results. Green color means that the web services worked properly and red color shows which web services failed.

    In order to see the debug mode just click on the black box of each site and then click on the green or red text to see web services answers.

  • Available classifications

    Here you can choose which taxa classification is the standard for your portal, the other classifications will be also available but with a manual user change.
